# go-iot-dispatch
IoT Диспетчер (Система управления трекерами)

Архитектура
├── cmd/
│   └── iot-gateway/
│       └── main.go         # Запуск HTTP-сервера и TCP-сервера параллельно
├── internal/
│   ├── tcp/
│   │   ├── server.go       # Слушает net.Listen(), принимает подключения от девайсов
│   │   └── registry.go     # МЕНЕДЖЕР СЕССИЙ (хранит map[deviceID]net.Conn с mutex для защиты)
│   ├── http/
│   │   └── handler.go      # Принимает REST запросы (например, POST /device/{id}/command)
│   └── usecase/
│       └── control.go      # Бизнес-логика: берет команду из HTTP, ищет девайс в registry, шлет в TCP

Суть: Сервер, который поддерживает постоянную связь с GPS-трекерами по TCP и позволяет отправлять им команды через Web-интерфейс (REST).
  - TCP-протокол (Девайсы):
    - При подключении трекер шлет приветствие: AUTH:device_id_123\n.
    - Каждые 10 секунд трекер шлет координаты: PING:lat=55.75,lon=37.61\n.
    - Принимает от сервера команды: CMD:LOCK\n или CMD:UNLOCK\n.
  - REST API (Для фронтенда/клиента):
    - GET /api/v1/devices — получить список всех подключенных девайсов и их последние координаты (из Redis).
    - POST /api/v1/devices/{id}/command — отправить команду на устройство (JSON: {"command": "LOCK"}).
  - Слой данных:
    - Postgres: история всех команд и лог подключений.
    - Redis: текущий статус устройства (online/offline) и последние координаты (чтобы не спамить в Postgres каждую секунду).
